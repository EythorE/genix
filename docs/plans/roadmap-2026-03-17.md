# Genix Improvement Roadmap — 2026-03-17

Prioritized roadmap based on the kernel review, test coverage analysis,
documentation audit, and project plan review. Focuses on software-only
improvements (no real hardware required).

---

## Tier 0: Bug Fixes (Do First)

These are actual bugs found during the kernel review. Fix before adding
new features.

### 0.1 Fix do_spawn_fd PID Cast Overflow
- **File:** kernel/proc.c
- **Issue:** Negative error from do_spawn() wraps to out-of-bounds index
- **Effort:** 2 lines
- **Risk:** Memory corruption on spawn failure

### 0.2 Fix do_exec XIP Fallback Memory Leak
- **File:** kernel/exec.c
- **Issue:** Original allocation not freed when XIP fails and re-allocates
- **Effort:** 3 lines
- **Risk:** Silent memory leak on XIP→RAM fallback path

### 0.3 Fix fs_read/fs_write Integer Overflow
- **File:** kernel/fs.c
- **Issue:** `off + n` can overflow uint32_t
- **Effort:** 2 lines per function
- **Risk:** Incorrect read/write size on large offset+size

### 0.4 Fix kstack Canary Panic — Disable Interrupts
- **File:** kernel/proc.c
- **Issue:** Timer ISR fires during panic loop with corrupted state
- **Effort:** 1 line (inline asm)
- **Risk:** Cascading corruption masks root cause

### 0.5 Mask Internal Flags in F_GETFL
- **File:** kernel/proc.c
- **Issue:** Pipe endpoint bits leak to userspace
- **Effort:** 1 line (mask before return)

## Tier 2: Performance (Phase 9)

These are documented in `docs/plans/optimization-plan.md` with full
FUZIX source references and cycle-count analysis.

### 2.1 Division Fast Path (DIVU.W)
- **File:** libc/divmod.S
- **Speedup:** 2-5x for 16-bit divisors
- **Effort:** ~20 lines asm
- **Impact:** Every printf, every division in user programs

### 2.2 Assembly memcpy/memset (MOVEM.L)
- **File:** libc/string.c → libc/memops.S
- **Speedup:** 4x for aligned blocks ≥32 bytes
- **Effort:** ~40 lines asm
- **Impact:** Every file read, pipe transfer, exec load

### 2.3 SRAM 16-bit I/O
- **File:** pal/megadrive/sram.S
- **Speedup:** ~20x for disk I/O
- **Effort:** ~10 lines asm
- **Impact:** All filesystem operations on Mega Drive

### 2.4 Pipe Bulk Copy
- **File:** kernel/proc.c (pipe_read/pipe_write)
- **Speedup:** 2-4x for pipeline throughput
- **Effort:** ~15 lines (replace byte loop with memcpy)
- **Impact:** All shell pipelines

## Tier 3: ANSI Color Terminal

### 3.1 ANSI Escape Parser in TTY
- **File:** kernel/tty.c
- **Effort:** ~200 lines
- **Enables:** Color output from all programs (levee, grep --color, etc.)
- **Approach:** State machine parsing ESC[...m (SGR), ESC[...H (cursor),
  ESC[J/K (clear), ESC[A-D (cursor movement)
- **Independent value:** Even without curses, this makes the VDP console
  much more useful

### 3.2 VDP Palette Setup for ANSI Colors
- **File:** pal/megadrive/vdp.S
- **Effort:** ~20 lines
- **Approach:** Fill 13 unused palette 0 slots with 8 ANSI colors.
  Use palette 3 for 8 bright/bold variants. OR current_attr into tile
  index before plotting.

### 3.3 Minimal Curses Library
- **File:** libc/curses.c (new)
- **Effort:** ~300-500 lines
- **API:** initscr, endwin, move, addch, addstr, attron/off, clear,
  clrtoeol, refresh, getch, init_pair, COLOR_PAIR
- **Approach:** ANSI escape output via TTY (Option A from VDP research).
  Portable across workbench and Mega Drive. NOT direct VDP access.
- **Enables:** tetris, snake, adventure, editors with status lines

## What We're NOT Doing

- **Complex VDP emulation** — BlastEm exists for this
- **Full ncurses port** — Too large (~50 KB). Minimal ANSI-based curses is sufficient
- **Hardware-dependent phases** (7, 8) — Need real EverDrive hardware
- **CI pipeline** — Nice to have but not blocking development
- **Multi-user support** — By design decision, single-user only
- **fork()** — No MMU, vfork+exec only
- **MMU emulation** — Not available on 68000
